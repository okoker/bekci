mechanism 01

        
Corroboration

        
Combine checks with boolean AND/OR groups, e.g. (TCP AND HTTP) OR PING. An alert fires only when independent signals agree the target is down. Groups nest and compose.

      

      

        
mechanism 02

        
Sustained failure

        
A condition must fail N times within a window before it counts as down (window minimum = fail_count × interval). One blip resets the streak and never pages.

      

      

        
mechanism 03

        
Flap suppression

        
Per-rule alert cooldown, a configurable re-alert interval for issues still firing, and a recovery cooldown — so a flapping target can never start an alert storm.

Eight ways to probe a target.

      
Each check yields a clean pass/fail — then conditions add confirmation and compose into the AND/OR groups above.

    


    

      
bekci@watch:~$ bekci checks --list # 8 probe types available

      

        
01ping/icmpReachability — reports RTT and packet loss. Passes on anything short of 100% loss.

        
02http/httpsExpected status code on a path or custom port; GET follows redirects.

        
03tcpStraight port-connect check — is the socket accepting?

        
04dnsResolve a host; optionally assert an expected record value, type and nameserver.

        
05page-hashSHA-256 of the response body — catches defacement and unexpected change.

        
06tls-certCertificate expiry — warn N days ahead of the deadline.

        
07snmp-v2csysUpTime, sysDescr, best-effort CPU and memory.

        
08snmp-v3Same metrics with authPriv / authNoPriv / noAuthNoPriv.

      

    

  







  

    

      
watch@bekci:~$ bekci features --all

      
Built by security people.

      
For teams tired of alert noise. Everything ships in the single binary.

    


    

      

        
/soc

        
SOC status wall

        
A flat, glanceable status page built for a security-operations display. Make it public if you want a shared board.

      

      

        
/sla

        
SLA compliance

        
Per-category SLA thresholds backed by 90-day daily-uptime charts, so you can prove the number.

      

      

        
/rbac

        
Role-based access

        
Three roles — admin, operator, viewer. JWT in an HttpOnly cookie; passwords hashed with bcrypt.

      

      

        
/alert

        
Alerting

        
Email (Resend or Microsoft 365), Signal, and a generic JSON webhook (Bearer or Basic). Recovery notes include downtime duration.

        email · signal · webhook
      

      

        
/backup

        
Encrypted backups

        
Full DB snapshots, optionally AES-256-GCM with an Argon2id-derived diceware passphrase. Config and full-DB restore.

      

      

        
/self

        
Dead man's switch

        
A scheduler heartbeat: the monitor watches itself and alerts if it ever stalls. Who watches the watchman.

      

      

        
/audit

        
Audit log

        
A full trail of every admin and operator action, with server-side search when you need to reconstruct what happened.

      

      

        
/f2b

        
fail2ban integration

        
Login brute-force protection with ban detail surfaced right in the console.
