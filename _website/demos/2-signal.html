01 Tactical two timescales, one glance

Two instruments, side by side.

Every check is read on two clocks at once. The 4-hour strip shows what’s happening right now; the 90-day strip shows whether it’s a pattern. An operator sees the live incident and the long trend without switching views.

4-hour · tactical 48 × 5 min = 4 h
What’s happening right now

48 segments, one every 5 minutes. Catches the blip, the recovery, the flap — the shape of the last few hours.

90-day · strategic 90 × 1 day = 90 d
Whether it’s a pattern

90 segments, one per day. Surfaces the slow degradation, the recurring Tuesday outage, the quarter-long trend.

Segment key 100% healthy 95–99% 80–94% <80% down no data
02 Zero false positives corroboration · sustained · suppressed

Most monitors alert on a single failed check. Bekci won’t.

It demands corroboration and sustained failure before it counts a target as down — three mechanisms working together to drive false positives toward zero.

Condition syntax
(TCP AND HTTP) OR PING

Groups combine by AND / OR internally; across groups the logic is OR. Nestable and composable.

01

Corroboration

Combine checks with boolean logic so an alert fires only when independent signals agree. Groups are joined by AND / OR — e.g. (TCP AND HTTP) OR PING — and nest freely into composable condition trees.

02

Sustained failure

A condition must fail N times within a window before it counts as down. A single blip is ignored. The window minimum is fail_count × interval, so you tune exactly how much evidence a page requires.

03

Flap suppression

Per-rule alert cooldown, a configurable re-alert interval for issues still firing, and a recovery cooldown — so a flapping target can’t turn one bad night into an alert storm.

03 Check types eight probes

Eight ways to ask “is it healthy?”

From an ICMP ping to an SNMPv3 read, every probe reports against explicit health criteria — then composes into the AND / OR logic above.

01PingICMPReachability, round-trip time and packet loss.
02HTTP / HTTPSHTTP·SExpected status code, endpoint path and custom port.
03TCPTCPRaw port connect — is the service accepting sockets?
04DNSDNSResolve a host; match record type, expected value and nameserver.
05Page HashSHA-256Hashes the response body to catch defacement or unexpected change.
06TLS CertificateTLSCertificate expiry — warn N days ahead of the deadline.
07SNMP v2cSNMP 2csysUpTime, sysDescr, best-effort CPU and memory.
08SNMP v3SNMP 3Same reads, secured with authPriv / authNoPriv / noAuthNoPriv.

›Each condition is field · comparator · value — status code match, DNS value, cert days-remaining, page-hash change — composed into the AND / OR groups above.

04 Capabilities operate · comply · secure

Everything the watch needs.

Bekci is a full operations surface, not just a prober — built with the controls a security team expects to find.

Alerting

Reach on-call three ways

Email (Resend or Microsoft 365), Signal, or a generic JSON webhook to any SOAR or chat endpoint — with Bearer or Basic auth. Recovery notes include the full downtime: down since, recovered, total.

EmailSignalWebhook
SOC

Status wall

A flat, glanceable status page built for the ops-room display. Make it public when you want a shareable one.

SLA

SLA compliance

Per-category SLA thresholds with 90-day daily-uptime charts to prove — or contest — the number.

RBAC

Roles & access

Admin, operator, viewer. JWT in an HttpOnly cookie, passwords hashed with bcrypt.

Backup

Encrypted backups

Full DB snapshots, optionally AES-256-GCM sealed with an Argon2id diceware passphrase. Automated, with config or full-DB restore.

Health

Self-health switch

A dead-man’s switch: the scheduler watches its own heartbeat and pages you if the monitor itself stalls.

Audit

Audit log

A full trail of every admin and operator action, with server-side search when you need to reconstruct events.

Security

fail2ban

Login brute-force protection with per-ban detail, so the front door isn’t the weak point.

05 Deploy one container · one port

Up in three commands.

One container, one binary, one port. No agents to roll out, no database to stand up — the Vue interface is embedded in the Go binary and SQLite lives on disk.

Then open http://localhost:65000.

Docker-first Single binary Port 65000 MIT licensed
quick start · docker 
$ git clone https://github.com/okoker/bekci.git
$ cd bekci
$ docker compose up -d
