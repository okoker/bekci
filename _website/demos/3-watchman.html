The signature readout

Two timescales, one glance.

Every check is drawn on two strips at once. The operator reads the incident happening right now and the ninety-day pattern behind it without switching a single view.

4h
The tactical view — last 4 hours
48 segments, one every 5 minutes. This is what's happening right now: the blip, the flap, the outage still in progress.
90d
The strategic view — last 90 days
90 segments, one per day. The long memory: reliability trend, recurring faults, and whether today's incident is an anomaly or a habit.

The pitch · zero false positives

Three witnesses must agree.

Most uptime monitors raise the alarm on a single failed check — and so most uptime monitors cry wolf. Bekci demands corroboration and sustained failure before it wakes anyone. Three independent guards, driving false positives toward zero.

Corroboration rule (TCP AND HTTP) OR PING

First witness

Corroboration

Combine checks with boolean logic so an alert fires only when independent signals agree. Groups join by AND/OR inside, OR across — nestable and composable.

Second witness

Sustained failure

A condition must fail N times within a window before it counts as down. A single blip is ignored; the minimum window is fail_count × interval.

Third witness

Flap suppression

Per-rule alert cooldown, a configurable re-alert interval for issues still firing, and a recovery cooldown — so a flapping target can never raise an alert storm.

The instruments

Eight ways to keep watch.

Each check evaluates health as field · comparator · value, and any of them can be composed into the corroboration groups above.

Ping

ICMP

Reachability with round-trip time and packet loss.

HTTP / HTTPS

Web

Expected status code, endpoint path and custom port.

TCP

Port

Confirms a TCP port accepts connections.

DNS

Resolve

Resolve a host, match record type, value and nameserver.

Page Hash

SHA-256

Hashes the response body to catch defacement or drift.

TLS Certificate

Expiry

Watches certificate expiry and warns N days ahead.

SNMP v2c

Community

sysUpTime, sysDescr, best-effort CPU and memory.

SNMP v3

authPriv

Same signals, with authPriv / authNoPriv / noAuthNoPriv.

Beyond the watch

Everything a security team runs on.

Bekci was built by security people, for people tired of alert noise — so the platform around the readout is built for a SOC, not a status badge.

Display

SOC status wall

A flat, glanceable status page built for a security-operations display — optionally made public.

Reliability

SLA compliance

Per-category SLA thresholds with 90-day daily-uptime charts to prove the numbers.

Access

RBAC · three roles

admin · operator · viewer, backed by JWT in an HttpOnly cookie and bcrypt hashing.

Alerting

Email · Signal · Webhook

Email via Resend or Microsoft 365, Signal REST, or generic JSON webhook. Recovery notices carry the full downtime duration.

Backups

Encrypted snapshots

Full-DB snapshots with optional AES-256-GCM (Argon2id, diceware passphrase). Automated, with config and full restore.

Resilience

Dead man's switch

A scheduler heartbeat: the monitor watches itself and alerts if its own watch ever stalls.

Trail

Audit log

A full trail of every admin and operator action, with server-side search.

Defense

fail2ban integration

Login brute-force protection, with per-ban detail surfaced in the interface.

Footprint

One binary, one port

A single Go binary with the Vue UI embedded, SQLite in WAL mode. No sprawling stack to guard.

Stand up the watch

Three commands, then it's awake.

Clone, compose, open. One container binds a single port and the owl starts counting heartbeats.

One container. No orchestration, no external database — SQLite in WAL mode ships inside.
One port. Everything answers on :65000 — UI and API alike.

bash — deploy the watch 
# clone the repository
$ git clone https://github.com/okoker/bekci.git
$ cd bekci

# one container, one port
$ docker compose up -d

# then open the watch station
→ http://localhost:65000
