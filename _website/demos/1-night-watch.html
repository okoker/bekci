Dual timescale

Two clocks, one glance.

Every check reports on two strips at the same time. The 4-hour strip breaks the last four hours into 48 five-minute segments — the tactical view, what's on fire right now. The 90-day strip lays out 90 daily segments — the strategic view, the reliability trend behind the moment. One row tells you both.

Signal integrity

Alert only when it's real.

Most uptime monitors page you on a single failed check. Bekci demands corroboration and sustained failure before it wakes anyone — driving false positives toward zero.

Corroboration rule
(TCPANDHTTP)ORPING

Fire only when TCP and HTTP both agree the host is down — or when a plain ping can't reach it at all. Groups combine by AND/OR inside; across groups the logic is OR. Nestable and composable.

Mechanism 01

Corroboration

Combine checks with boolean AND/OR condition groups. An alert fires only when independent signals agree — one flaky probe can't speak for the whole host.

Mechanism 02

Sustained failure

A condition must fail N times within a fail window before it counts as down. A single blip is ignored. The window is at least fail count × interval.

Mechanism 03

Flap suppression

Per-rule alert cooldown, a configurable re-alert interval for issues still firing, and a recovery cooldown — so a flapping target can't spawn an alert storm.

Check types

Eight ways to ask “are you there?”

From a raw ICMP echo to an authenticated SNMPv3 poll. Each condition is a field, a comparator and a value — composable into the corroboration groups above.

Deploy

One container. One binary. One port.

root@watchtower : ~
$ git clone https://github.com/okoker/bekci.git
$ cd bekci
$ docker compose up -d
⠿ Container bekci Started
# listening on :65000 — the watch is on.

A single Go binary with the Vue 3 UI embedded, SQLite for storage, Docker-first. No agents to install, no external database, no message bus. It scales to roughly 1,000 hosts on one small box.

Then open http://localhost:65000 and sign in.
